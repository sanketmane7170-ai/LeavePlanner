# Deploying the Leave Planner alongside an existing project

This kit deploys the Leave Planner (project #2) on a server already running
another app, **without touching project #1**. Isolation guarantees:

| Resource        | Value for this project                          |
|-----------------|-------------------------------------------------|
| Code            | `/var/www/leave-planner`                        |
| Backend port    | `127.0.0.1:4001` (localhost only)               |
| Frontend port   | `127.0.0.1:4003` (localhost only)               |
| PM2 apps        | `leave-backend`, `leave-frontend`               |
| Database        | `innovizia_leave_planner` + user `leaveplanner` |
| Nginx           | a **new** file in `sites-available`             |
| Logs            | `/var/log/leave-planner/`                       |

Only nginx faces the internet; both Node apps bind to localhost. Project #1's
ports, database, PM2 apps, and nginx files are never modified.

---

## Prerequisites

- A domain or subdomain (e.g. `leave.yourcompany.com`) with an **A record →
  `103.212.120.17`**. HTTPS (and therefore working logins) needs this.
  - No domain yet? You can test over plain HTTP first by setting
    `COOKIE_SECURE=false` in `backend/.env` and visiting `http://SERVER_IP`,
    but switch to a domain + HTTPS before real use.

---

## Step 1 — One-time setup

SSH in as root, then:

```bash
curl -fsSL https://raw.githubusercontent.com/sanketmane7170-ai/LeavePlanner/main/deploy/setup.sh -o /tmp/lp-setup.sh
bash /tmp/lp-setup.sh
```

This runs **read-only pre-flight checks first** and aborts if ports 4001/4003,
the PM2 names, or the database already exist — so it can never clobber
project #1. It then installs any missing tooling, creates the isolated DB,
clones the repo to `/var/www/leave-planner`, and writes `backend/.env` with a
generated DB password + JWT secret.

## Step 2 — Fill in the env

```bash
nano /var/www/leave-planner/backend/.env
```
Set: `FRONTEND_URL` (your https domain), `ADMIN_EMAIL`, `ADMIN_PASSWORD`,
`ADMIN_NAME`, and the `SMTP_*` values. Leave `COOKIE_SECURE=true` (HTTPS).

## Step 3 — Build & start

```bash
bash /var/www/leave-planner/deploy/deploy.sh
```
Builds backend + frontend, creates the DB schema (`prisma db push`), seeds the
first admin, and starts both apps under PM2. Re-run this same script for every
future update (it does `git pull` → rebuild → zero-downtime `pm2 reload`).

Make PM2 survive reboots (safe — preserves project #1's PM2 list too):
```bash
pm2 startup systemd -u root --hp /root   # run the line it prints
pm2 save
```

## Step 4 — Nginx (new file, never edits project #1)

```bash
cp /var/www/leave-planner/deploy/nginx-leave-planner.conf \
   /etc/nginx/sites-available/leave-planner.conf

# set your real domain
nano /etc/nginx/sites-available/leave-planner.conf   # replace leave.example.com

ln -s /etc/nginx/sites-available/leave-planner.conf \
      /etc/nginx/sites-enabled/leave-planner.conf

nginx -t            # MUST pass — validates ALL configs incl. project #1
systemctl reload nginx   # reload, not restart → project #1 never drops a request
```

## Step 5 — HTTPS (Let's Encrypt)

```bash
apt-get install -y certbot python3-certbot-nginx
certbot --nginx -d leave.yourcompany.com
```
Certbot edits **only this project's** server block to add the 443 listener and
HTTP→HTTPS redirect, and sets up auto-renewal.

---

## Verifying it works

```bash
pm2 ls                                   # leave-backend + leave-frontend = online
curl -s http://127.0.0.1:4001/health     # {"status":"ok","db":"connected"}
curl -sI http://127.0.0.1:4003           # 200 from Next.js
```
Then open `https://leave.yourcompany.com` and log in with the admin credentials
from `backend/.env`.

## Updating later

```bash
bash /var/www/leave-planner/deploy/deploy.sh
```

## Rollback / stop (project #1 unaffected)

```bash
pm2 stop leave-backend leave-frontend
rm /etc/nginx/sites-enabled/leave-planner.conf && nginx -t && systemctl reload nginx
```

---

## Security reminders

- Rotate the root SSH password you shared, and switch to SSH key auth.
- Rotate the SMTP token and OpenAI key before production use.
- The DB password and JWT secret are auto-generated by `setup.sh` and live only
  in `backend/.env` (gitignored).
